### モダン JavaScript
  - **ES2015(ES6)以降の新しい構文や機能**
    - const/let による変数宣言
    - アロー関数
    - テンプレートリテラル
    - 分割代入
    - スプレッド構文
    - クラス構文
    - Promise を使った非同期処理
  - **モジュールシステムの採用**
    - import/export を使ったモジュール化
    - webpack などのモジュールバンドラーの利用
      - モジュールバンドラー
        - 複数の JavaScript ファイルやモジュールを 1 つのファイルにまとめる役割を持つ
        - 特徴
          - 依存関係の解決: 複数のファイル間の依存関係を自動的に解析し、適切な順序でコードをバンドルする
          - パフォーマンスの向上: 複数のファイルを 1 つにまとめることで、HTTP/1.1 での通信回数を減らし、ページの読み込み速度を向上させる
          - 開発効率の向上: 開発者は機能ごとに分割されたモジュールで作業でき、コードの管理や再利用が容易になる
          - 名前空間の汚染防止: モジュール化により、変数名の競合を避けることができる
        - 例
          - Webpack
          - Rollup
          - Parcel
  - **仮想 DOM を用いたライブラリ/フレームワーク**
    - React、Vue、Angular など
  - **ツールチェーンの活用**
    - npm/yarn によるパッケージ管理
    - Babel によるトランスパイル
      - トランスパイル
        - あるプログラミング言語で書かれたコードを別の言語や同じ言語の異なるバージョンに変換するプロセス
        - 特徴
          - 最新の構文対応
            - ES6+で書かれたコードを、古いブラウザでも動作する ES5 に変換
          - 型チェック
            - TypeScript のようなスーパーセット言語から JavaScript への変換を行う
          - 新機能
            - 最新の JavaScript 機能を使用しつつ、広範なブラウザ互換性を維持
        - 例
          - Babel
    - ESLint による静的解析
  - **新しい開発手法**
    - SPA 開発
      - Single Page Application
      - 特徴
        - 単一のWebページで構成される
        - ページ遷移を行わずにコンテンツを動的に更新する
        - JavaScriptを使用してHTMLの一部を差し替える
      - 動作の仕組み
        1. 初回アクセス時に1つのHTMLページを読み込む
        2. ユーザーの操作に応じて必要な部分のみを非同期で更新する
        3. サーバーとの通信はAPIを介して行い、JSONなどのデータのみをやり取りする
      - メリット
        - 高速な操作性とスムーズなユーザー体験を提供
        - サーバーとの通信量を削減し、パフォーマンスを向上
        - ネイティブアプリに近い使用感を実現
      - 代表的な実装例
        - Googleマップ
        - Facebookメッセンジャー
        - Gmail
      - 開発に使用されるフレームワーク
        - React
        - Vue.js
        - Angular
      - 従来の開発手法との違い
        - 画面の生成方法
          - 従来のWebアプリ
            - サーバー側で完全なHTMLを生成し、クライアントに送信する
          - SPA
            - クライアント側でJavaScriptを使用して動的にHTMLを生成する
        - ページ遷移
          - 従来のWebアプリ
            - ユーザーの操作ごとに新しいHTMLをサーバーから取得し、ページ全体を再読み込む
          - SPA
            - 単一のHTMLページ内で、JavaScriptによってコンテンツを動的に更新する
        - サーバーとの通信
          - 従来のWebアプリ
            - 各ページ遷移でHTMLを含む完全なレスポンスを受け取る
          - SPA
            - 必要なデータのみをJSON形式で非同期に取得する
        - 初期ロード時間
          - 従来のWebアプリ
            各ページで必要な分だけ読み込むため、初期ロードは比較的速い
          - SPA
            アプリケーション全体のリソースを最初に読み込むため、初期ロードに時間がかかる場合がある
        - ユーザー体験
          - 従来のWebアプリ
            - ページ遷移ごとに画面全体が更新されるため、操作感が遅く感じられる場合がある
          - SPA
            - 画面の一部のみを更新するため、よりスムーズで高速な操作感を提供する

    - コンポーネント指向の設計

### DOM
  - Document Object Model
  - HTMLやXMLなどのマークアップ言語で作成されたドキュメントをプログラムから操作するための仕組み
  - 特徴
    - 階層構造（ツリー構造）
      - DOMはドキュメントの構造をツリー状の階層構造として表現
      - ルートノード（html要素）を頂点とし、その下に子ノードが続く
      - 各要素（タグ）はノードとして扱われる
      - ノード間の関係は親子関係や兄弟関係として表現される
    - プログラムからの操作
      - JavaScriptなどのプログラミング言語からDOMを介してHTMLを操作できる
      - 要素の取得、追加、削除、変更が可能
      - スタイルの変更やイベントの追加なども行える
    - 主要なインターフェース
      - DOMには以下のような主要なインターフェースがある
        - Document
          - ドキュメント全体を表す
        - Element
          - HTML要素を表す
        - Node
          - すべてのノードの基本となるインターフェース
    - DOMの役割
      - HTMLとJavaScriptを橋渡しする
      - ウェブページの動的な操作を可能にする
      - インタラクティブなウェブアプリケーションの開発に不可欠

### 仮想DOM
  - Virtual DOM
  - 実際のDOMを抽象化した軽量なJavaScriptオブジェクト
  - 特徴
    - メモリ上に存在する仮想的なDOM表現
    - 実際のDOMの軽量なコピーとして機能する
    - ReactやVue.jsなどのフロントエンドフレームワークで使用される
  - 仕組み
    - DOM操作が発生すると、新しい仮想DOMを生成する
    - 新旧の仮想DOMを比較し、差分を検出する（差分検出処理）
    - 検出された差分のみを実際のDOMに反映させる
  - 利点
    - DOM操作の効率化
      - 必要最小限の更新のみを行うため、パフォーマンスが向上する
    - バッチ処理
      - 複数の変更をまとめて実行できる
    - 開発の容易さ
      - 複雑なDOM操作の管理を自動化する
  - 実際のDOMとの違い
    - 実際のDOMは直接ブラウザに表示されますが、仮想DOMはメモリ上にのみ存在する
    - 仮想DOMは実際のDOMより軽量で、操作が高速

### ECMAScript
  - ECMA
    - European Computer Manufacturers Association
      - 欧州電子計算機工業会
      - 1994年に名前が変わり、Ecma Internationalとなる
      - 主な標準
        - ECMAScript (JavaScript の標準仕様)
        - JSON (JavaScript Object Notation)
        - C# 言語仕様
  - JavaScriptの標準仕様を定義したもの
  - 特徴
    - 標準化された言語仕様
      - ECMAScriptは、JavaScriptの中核となる言語機能を標準化している
    - 開発と管理
      - Ecma International（旧欧州電子計算機工業会）によって開発・管理されている
    - 定期的な更新
      - 1997年の初版以降、定期的にバージョンアップが行われており、現在は毎年6月に新バージョンがリリースされている
    - JavaScriptとの関係
      - ECMAScriptはJavaScriptの基盤となる仕様ですが、完全に同一ではない
      - JavaScriptはECMAScriptに基づきつつ、ブラウザ固有の機能などを追加している
    - 言語の特徴
      - ECMAScriptは、プロトタイプベース、関数型、命令型などの複数のプログラミングパラダイムをサポートする多目的言語
    - 影響力
      - ECMAScriptはJavaScriptだけでなく、ActionScriptやJScriptなど他のスクリプト言語にも影響を与えている
    - 最新の機能
      - 最新のバージョンでは、非同期プログラミング（async/await）、モジュールシステム、クラス構文など、モダンな言語機能が導入されている
